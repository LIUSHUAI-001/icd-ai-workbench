## 2. ICD AI WORK 原项目可继承的布局原则

### 2.1 从 AppHeader 提取

- **品牌优先**：左侧 `p24-brand`（logo mark + 文字）固定在最左
- **导航居中**：`p24-nav` 水平排列，5 个主要入口（首页/画布/灵感库/案例导航/资产库）
- **单一 CTA**：右侧 `p24-button--nav`（"进入工作台"）
- **Header 紧凑**：单行，~52px 高，`backdrop-filter: blur` 毛玻璃背景
- **无冗余**：无推广链接、无广告、无彩蛋

### 2.2 从 HomePage 提取

- **Hero → 项目索引 → 工作流 → Footer** 的垂直叙事结构
- **项目统计三数字**：项目总数 / 活跃数 / 工作流数
- **最近打开**：单条最近项目入口
- **工作流卡片**：编号、kicker、标题、流程描述、箭头、状态指示
- **Footer**：简洁的产品签名行

### 2.3 从 styles.css 提取

- **设计令牌**：`--p24-black`(#000), `--p24-rail`(#292d30), `--p24-smoke`(#464a4d), `--p24-fog`(#a1a4a5), `--p24-frost`(#f0f0f0)
- **圆角系统**：卡片 16px，药丸 999px，按钮/输入框 6px
- **焦点环**：`rgba(59,158,255,0.66)` border + `rgba(59,158,255,0.09)` box-shadow
- **按钮体系**：primary（蓝底白字）、nav（蓝边框）、ghost（无边框）
- **过渡缓动**：`cubic-bezier(0.23, 1, 0.32, 1)`

### 2.4 ICD 布局原则总结

1. **极简顶栏** — 品牌 + 导航 + 一个 CTA，不要更多
2. **垂直叙事** — 页面内容自上而下讲述产品故事
3. **数字优先** — 用统计数据传达产品价值
4. **工作流驱动** — 用预设工作流降低新用户认知负担
5. **深色单色底** — 黑色/石墨底 + 蓝色/紫色功能色点缀

## 4. 左侧菜单/侧边栏分组结构（提案）

### 4.1 当前问题

- 侧边栏混合了画布管理和节点面板，但在 ICD 语境下没有「工作台」感
- 节点面板直接暴露了 ~40 种节点，新用户容易迷失
- 画布列表与节点面板之间没有清晰的分组标签

### 4.2 推荐分组（保持现有侧边栏 DOM，仅通过 CSS/文案调整）

```
┌─────────────────────────┐
│ 🔍 搜索画布…             │  ← 保留
├─────────────────────────┤
│ 📋 画布列表              │  ← 分组标签（新增 CSS）
│  · 画布 1               │
│  · 画布 2 (激活)        │
│  + 新建画布              │
├─────────────────────────┤
│ 🧩 节点面板              │  ← 分组标签（新增 CSS）
│  ▼ 核心节点              │
│    文本  图像  视频  LLM │
│  ▼ 输入输出              │
│    上传  输出  素材集    │
│  ▼ AI 工作流             │
│    RunningHub  RH超市    │
│    ComfyUI超市           │
│  ▼ 工具箱                │
│    画板  循环器  对比    │
│    抽帧  尺寸  宫格      │
├─────────────────────────┤
│ ICD STUDIO v2.3.8       │  ← 已经过 CSS 覆盖
└─────────────────────────┘
```

### 4.3 节点类别重命名提案

| 原 T8 category | 当前中文 | 提案 ICD 名称 | 理由 |
|---|---|---|---|
| `core` | 核心 | **AI 生成** | 更直观 |
| `input` | 输入 | **素材入口** | 避免与「输入框」混淆 |
| `rh` | RH | **RunningHub** | 保持产品名 |
| `comfyui` | ComfyUI | **ComfyUI** | 保持产品名 |
| `utility` | 工具 | **辅助工具** | 降低优先级感知 |
| `toolbox` | 工具箱 | **设计工具箱** | ICD 语境 |
| `auxiliary` | 辅助 | **其他** | 收纳低频节点 |

> **注意**：节点类别标签在 Sidebar.tsx 中渲染，纯 CSS 无法修改文字。重命名类别需要修改 Sidebar.tsx。当前阶段仅做提案，等待用户批准后再动代码。

## 9. 用户审批清单（5 个关键决策）

### 决策 1：顶栏保留哪些入口？

**默认建议**：仅保留 ICD STUDIO 品牌、资源库、API 设置。其余全部隐藏或收纳。

| 选项 | 说明 |
|---|---|

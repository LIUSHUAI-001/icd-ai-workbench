# 贞贞的无限画布 v2.4.1

本次 Electron 自动更新版聚焦扩展 API 平台下 Agnes AI 专用图像编辑的 503 修复。v2.4.0 已修复 OpenAI 兼容模式里的 Agnes 图像编辑；这一版把同样的官方请求格式补齐到 Agnes 专用适配器，避免图像编辑入口传入 `imageUrls` 等别名时参考图没有进入 `extra_body.image`。

## 主要更新

- Agnes AI 专用适配器继续使用 `/v1/images/generations` JSON 请求；图像编辑参考图统一放入 `extra_body.image`，`response_format` 放入 `extra_body.response_format`。
- 扩展平台图像编辑入口兼容 `images`、`image`、`imageUrl`、`image_url`、`imageUrls`、`image_urls`、`referenceImages`、`reference_images`，覆盖前端和路由里常见的参考图字段别名。
- 不向 Agnes 图生图请求发送顶层 `response_format` 或 `tags`；只有无参考图且显式请求 `b64_json` 时才开启 `return_base64`。
- 新增 Agnes adapter 单测和 `/api/proxy/external/image` 路由级回归，锁定专用 Agnes 图像编辑请求体。
- 已用真实 Agnes API 验证专用扩展平台文生图与 `imageUrls` 别名图像编辑均成功返回远端 URL，并由后端转存为本地输出文件。

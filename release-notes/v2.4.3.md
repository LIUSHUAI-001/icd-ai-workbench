# 贞贞的无限画布 v2.4.3

本次 Electron 自动更新版解除上传素材节点的 20MB 应用层大小限制。大体积本地图片、视频、音频或通用素材可以继续通过上传素材节点进入画布，实际容量只受磁盘、系统和运行环境约束。

## 主要更新

- 上传素材节点不再提示“文件超过上传上限 20MB，请压缩后重试”。
- 后端 `/api/files/upload` 移除 multer `limits.fileSize`，不再在应用层拒绝 20MB 以上文件。
- `backend/src/config.js` 保留 `MAX_FILE_SIZE: 0` 作为“不设置应用层大小上限”的明确语义。
- 上传错误处理移除 `LIMIT_FILE_SIZE` / `file_too_large` 专用分支，避免继续把大文件误判为 20MB 超限。
- 回归测试锁定上传路由不再包含 fileSize 限制、20MB 超限文案和旧错误码。

- 公开版本号同步到 `2.4.3`：`package.json`、`package-lock.json`、`vite.config.ts`、`vite.config.js`、`electron/main.cjs`、`backend/src/config.js`、`features.json`、`README.md` 与本 release notes 同步。
- 已做真实后端上传冒烟：临时端口 POST `/api/files/upload` 上传 21MB 文件返回 `200 success`，并清理测试生成的 input 临时文件。
